# Real-Time Log Analytics Pipeline

A lightweight end-to-end observability demo that streams synthetic application logs through Kafka into ClickHouse and visualizes service health in Grafana.

[![Python Tests](https://github.com/HarshalS007/real-time-log-analytics-pipeline/actions/workflows/python-tests.yml/badge.svg)](https://github.com/HarshalS007/real-time-log-analytics-pipeline/actions/workflows/python-tests.yml)

This project demonstrates a real-time log analytics pipeline with:

- 50K+ events per minute generated by synthetic microservices
- Kafka-based ingestion in near real time
- Direct ClickHouse integration using the Kafka table engine and materialized views
- Grafana dashboards for P95 latency, error rates, throughput, and incident visibility

```text
Producer → Kafka → ClickHouse → Grafana
```

## Why this project exists

The goal is to show how a modern observability stack can be assembled locally without a full production environment. The synthetic incident simulator makes the dashboards meaningful by introducing short service degradations that look realistic.

## How it works

1. A Python producer generates synthetic log events.
2. Kafka receives the event stream.
3. ClickHouse ingests the data directly through its native Kafka engine.
4. Materialized views transform the stream into analytics-friendly tables.
5. Grafana turns the resulting data into live operational dashboards.

## Tech stack

- Python
- Kafka
- ClickHouse
- Grafana

## Quick start

You need Docker and Docker Compose installed.

```bash
docker compose up --build
```

This brings up the full stack in order:

1. Kafka
2. ClickHouse
3. Grafana
4. The producer that begins streaming synthetic logs

Then open:

- Grafana: http://localhost:3000
  - username: admin
  - password: admin
- ClickHouse HTTP console: http://localhost:8123/play

## Project structure

- producer/ — Python generator and sink logic
- clickhouse/init/01_create_tables.sql — ClickHouse schema and views
- grafana/ — datasource provisioning and dashboard definition
- docker-compose.yml — starts the full local stack

## Quick checks

Confirm that data is flowing by querying ClickHouse:

```bash
curl -s 'http://localhost:8123/?query=SELECT+count()+FROM+logs_db.logs+WHERE+event_time+>=+now()+-+INTERVAL+1+MINUTE'
```

For a quick latency view:

```bash
curl -s 'http://localhost:8123/?query=SELECT+quantile(0.50)(latency_ms),+quantile(0.95)(latency_ms),+max(latency_ms)+FROM+logs_db.ingestion_latency+FORMAT+TSV'
```

## Run the producer on its own

If you want to test the log generator without Docker:

```bash
cd producer
pip install -r requirements.txt
python producer.py --dry-run --rate 50 --duration 10
python test_log_events.py
```
